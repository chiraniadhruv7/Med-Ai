Multimodal Clinical AI Assistant

"Can a multimodal AI that fuses CNN image analysis with medical LLM reasoning provide more reliable, explainable, and actionable chest X-ray diagnostic support than single-modality approaches?"

Problem Statement

Over 2 billion chest X-rays annually, yet radiologist access is limited globally. Fatigue errors: 3-5%. Existing AI: single-modality, no explanations, no clinical context, no confidence calibration, no cross-validation between image findings and patient symptoms.

Literature Review

  • • CheXNet (2017) - DenseNet121, radiologist-level, no explainability
  • • CheXpert (2019) - Uncertainty labels, single-modality
  • • MedGemma (2025) - Medical LLM, no CNN fusion
  • • Grad-CAM (2017) - Visual explanations, not in clinical pipelines

Research Gap

  • • No CNN + LLM fusion with contradiction detection
  • • No automated disagreement penalty
  • • No multi-source calibrated uncertainty
  • • Our: 7-stage end-to-end pipeline

Fusion Layer

  • • 60% CNN + 40% LLM weighted
  • • Agreement: 1.1x boost
  • • Contradiction: 0.75x penalty
  • • Conf = 0.4C + 0.3L + 0.3A
  • • Each contradiction: -5%

Why DenseNet121 + MedGemma 4B?

DenseNet121 vs Others

Dense Connectivity

Every layer connects to all others. Feature reuse preserves subtle medical findings without gradient loss.

Medical Gold Standard

CheXNet (Stanford 2017) used DenseNet121 for radiologist-level chest X-ray analysis. Remains the benchmark.

8M Parameters

vs ResNet50 (25M), ViT-B (86M). Fits T4 alongside MedGemma 4B. Less overfitting on medical data.

Superior Grad-CAM

Dense blocks produce rich feature maps. Heatmaps are sharper, more clinically meaningful than ResNet or ViT.

MedGemma 4B vs Others

Purpose-Built Medical

Trained on medical imaging + clinical text. GPT-4V is general-purpose. BiomedCLIP is embeddings-only.

Native Multimodal

Image + text in single pass. No separate encoders or CLIP alignment needed.

Free + Open Weight

4B params in bf16 fits T4. GPT-4V costs $$$. Med-PaLM 2 is closed-source.

Structured Output

Generates observations, differentials, follow-up in parseable sections for fusion.

How It Works

  1. Extract feature maps from last DenseNet block
  2. Backpropagate gradients for target pathology
  3. Global average pooling for channel weights
  4. Weighted sum + ReLU activation
  5. Bilinear upsample to 224x224, overlay at 40%
